### Cesium.CustomDataSource
```js
let shipTrack = new Cesium.CustomDataSource(`ship_his`);
```

- 这行代码的作用是新建一个名为 ship_his 的自定义数据源，用于后续添加船舶历史轨迹的实体（如点、线等）。

- Cesium.CustomDataSource 是 Cesium 提供的一个容器，可以批量管理一组实体（Entity），并通过名字进行统一增删。

- 这样做的好处是便于后续对整组船舶轨迹进行统一管理（如显示、隐藏、删除等）。

```js
this.removeCustom(`ship_his`);
```
- 这行代码的作用是移除名为 ship_his 的自定义数据源（CustomDataSource）。

- removeCustom 很可能是组件/类中的一个方法，用于从 Cesium 的 viewer.dataSources 中查找并删除指定名称的数据源。

- 这样做的目的是先清理旧的“船舶历史轨迹”图层，避免重复叠加或数据混乱。
### Cesium.Viewer
![image.png](https://s2.loli.net/2025/08/05/2bDtpSy8CINwZvl.png)

```js
this.viewer = new Cesium.Viewer('cesiumContainer', {
    geocoder: false, // 右上角查询搜索（关闭地址/地名搜索框）
    homeButton: false, // home按钮（关闭返回初始视角的按钮）
    timeline: false, // 时间轴（关闭底部时间轴控件）
    animation: false, // 左下角仪表（关闭动画控制器）
    sceneModePicker: true, // 3d 2d选择器（允许用户切换2D/3D视图）
    baseLayerPicker: true, // 图层选择器（允许用户切换底图图层）
    navigationHelpButton: false, // 右上角帮助按钮（关闭导航帮助）
    selectionIndicator: false, // 选中框（关闭实体选中高亮框）
    infoBox: false, // 信息框（关闭实体信息弹窗）
    // scene3DOnly: true // （注释掉，若启用则只允许3D模式）
});
```
简要说明：
- Cesium.Viewer 是 CesiumJS 的核心视图组件，负责渲染地图和交互。
- 'cesiumContainer' 是页面上用于承载地图的 DOM 元素 id。
- 配置项通过布尔值控制各个内置 UI 组件的显示与否，目的是让地图界面更简洁，只保留必要的功能（如图层切换和2D/3D切换），去除不需要的控件。
总结：
这段代码的作用是创建一个定制化的 Cesium 地图视图，隐藏了大部分默认控件，仅保留图层选择和2D/3D切换功能，使界面更简洁。
