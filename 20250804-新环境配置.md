## 方法一：PowerShell 命令行快速安装 WSL2（推荐）

1. 启用 Windows 子系统（WSL）功能
在 PowerShell（管理员模式）中运行：
``` powershell
dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart
```

2. 启用虚拟机平台功能

WSL2 依赖 Windows 虚拟机功能，需要额外启用：
```powershell
dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart
```

3. 将 WSL 默认版本设置为 WSL2

```powershell
wsl --set-default-version 2
```

> - Windows 11 默认已安装 WSL2，不需要执行该命令。
> - Windows 10 用户 需要执行此命令，否则默认使用 WSL1。

4. 重启电脑
以上命令执行完成后，请重启电脑，以确保所有更改生效。

## 查看可用的wsl版本

``` bash
wsl --list --online
```

## 安装ubuntu
``` bash
// 安装默认版本
wsl --install

// 安装指定版本
wsl --install -d Ubuntu-20.04
```

安装完毕后会要求你创建一个新用户，按照提示输入用户名和密码即可

# git config 配置

```
git config --global pull.rebase false
git config --global credential.helper store
```
还原配置:
## 1.删除全局凭据存储
```bash
git config --global --unset credential.helper
```

这会移除全局的 credential.helper store 配置。
## 2. 删除存储的凭据文件

凭据存储文件通常位于：

- Windows: %USERPROFILE%\.git-credentials

- Linux/macOS: ~/.git-credentials

你可以直接删除这个文件：
```bash
# Windows
del %USERPROFILE%\.git-credentials

# Linux/macOS
rm ~/.git-credentials
```
## 3.完全重置全局配置
```
# 删除全局配置文件
# Windows
del %USERPROFILE%\.gitconfig

# Linux/macOS
rm ~/.gitconfig
```
